# AGENTS.md — 开发工作区使用说明

本文件供后续参与开发的 AI Agent 阅读，请严格遵守以下规范。

---

## 文件夹结构与用途

```
Template tool/
├── docs/              # 产品文档区
├── assets/
│   ├── design/        # 设计素材
│   ├── bug/           # 测试报错截图
│   └── reference/     # 参考图、灵感收集
├── notes/             # 学习笔记
└── src/               # 代码目录
```

### docs/

存放所有产品文档，包括但不限于：

- PRD（产品需求文档）
- 需求迭代记录
- 关键决策记录（ADR）
- API 文档、接口说明

**规则：** 所有 `.md`、`.docx`、`.pdf` 等文档类文件一律放在此目录，禁止散落在根目录或其他文件夹。

### assets/design/

存放 UI/UX 相关素材：

- 效果图、高保真设计稿
- UI 参考截图
- 组件规范图

### assets/bug/

存放测试阶段的报错截图和复现记录：

- Bug 截图以 `bug-<编号>-<简述>.png` 命名
- 可附带对应的 `.txt` 或 `.md` 说明文件

### assets/reference/

存放参考素材：

- 竞品截图
- 灵感收集
- 技术方案参考图

### notes/

存放开发过程中的学习笔记：

- 踩坑记录
- 技术方案对比
- 第三方库使用心得

**规则：** 笔记以日期或主题命名，方便检索。

### src/

存放所有代码文件。

**规则：**

- 所有源代码必须在此目录内，禁止在根目录或其他文件夹存放代码文件
- 代码中引用的资源（图片、配置等）应使用相对路径，且仅引用项目内部文件
- 禁止硬编码项目外部的绝对路径
- 第三方依赖通过包管理器管理，不要手动复制库文件到项目中

---

## 开发规则

1. **代码区隔离**：`src/` 是唯一的代码目录，所有代码逻辑、配置文件、脚本均放在此处。
2. **文档归档**：任何文档性质的文件（需求、方案、笔记）都放入 `docs/` 或 `notes/`，不要放在根目录。
3. **资源分类**：图片、截图等素材按类型放入 `assets/` 的对应子目录，不要混放。
4. **不引用外部文件**：代码中不得引用项目目录以外的文件路径，确保项目可移植。
5. **命名规范**：
   - 文件夹使用小写英文，多词用 `-` 连接（如 `my-component`）
   - 文件名使用小写英文或有意义的命名
   - 截图类文件带编号或日期前缀
6. **提交规范**：每次提交说明做了什么改动，便于追溯。
7. **新增文件前先检查**：创建新文件前确认它应该放在哪个目录，遵循上述分类。

- 如果需要新增文件夹分类，请同步更新本文件
