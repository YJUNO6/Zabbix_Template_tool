# 测试指南

## 测试环境准备

### 1. 启动服务
```bash
# 终端1: 启动后端
cd src/backend
pip install -r requirements.txt
python -m uvicorn app.main:app --port 8000

# 终端2: 启动前端
cd src/frontend
npm install && npm run dev
```

### 2. 准备测试MIB文件
准备一个标准的MIB文件 (如 Lenovo 服务器MIB)，确保包含:
- 独立静态OID (如整机健康状态、设备型号)
- 表格类索引OID (如CPU列表、风扇列表、电源列表)

| 检查项 | 验证方法 | 预期 |
|--------|---------|------|
| 状态类分类 | 查看含health/status的指标 | category=status, delay=1m |
| 性能类分类 | 查看含temperature/power的指标 | category=performance, delay=5m |
| 信息类分类 | 查看含model/version的指标 | category=info, delay=1d |

## API接口测试

使用 curl 或 http://localhost:8000/docs 的Swagger UI 测试:

```bash
# 1. 上传文件
curl -X POST http://localhost:8000/api/upload \
  -F "files=@test.mib"

# 2. 解析MIB
curl -X POST http://localhost:8000/api/parse \
  -H "Content-Type: application/json" \
  -d '{"filenames": ["test.mib"], "vendor_oid": "1.3.6.1.4.1.53184"}'

# 3. 获取预览
curl http://localhost:8000/api/preview/{task_id}

# 4. 导出XML
curl -X POST http://localhost:8000/api/export \
  -H "Content-Type: application/json" \
  -d '{"task_id": "xxx", "config": {...}}' \
  -o template.xml

# 5. 查看翻译词库
curl http://localhost:8000/api/translations

# 6. 查看历史任务
curl http://localhost:8000/api/tasks
```

## 已知限制

1. MIB文件需要是标准SMIv1/SMIv2格式
2. 依赖的MIB文件(如SNMPv2-MIB)需要预装或放在搜索路径中
3. 极大的MIB文件(>10000 OID)解析可能较慢
4. 自动分类基于关键词匹配，可能需要人工微调
