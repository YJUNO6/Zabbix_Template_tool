# 使用说明

## 环境要求

| 软件 | 最低版本 | 下载地址 |
|------|---------|---------|
| Python | 3.9+ | https://python.org/downloads/ |
| Node.js | 16+ | https://nodejs.org/ |

安装完成后验证:
```cmd
python --version    # 应显示 3.9 以上
node --version      # 应显示 16 以上
```

## 一键启动 (推荐)

双击运行 `src/start.bat`，或在命令行执行:

```cmd
cd E:\dev\app\Template tool\src
start.bat
```

脚本会自动:
1. 检查 Python 和 Node 是否安装
2. 安装后端 Python 依赖
3. 启动后端服务 (端口 8000)
4. 安装前端 Node 依赖 (首次)
5. 启动前端服务 (端口 5173)

启动完成后会自动打开两个命令行窗口，不要关闭它们。

然后打开浏览器访问: **http://localhost:5173**

## 手动启动

如果 `start.bat` 有问题，打开两个命令行窗口分别执行:

**窗口1 - 后端:**
```cmd
cd /d "E:\dev\app\Template tool\src\backend"
pip install -r requirements.txt
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

**窗口2 - 前端:**
```cmd
cd /d "E:\dev\app\Template tool\src\frontend"
npm install
npm run dev
```

## 功能操作流程

### 第一步: 上传MIB文件
1. 打开首页 http://localhost:5173
2. 在左侧"文件上传区"拖入或选择 .mib / .txt 文件
3. (可选) 输入厂商前缀OID，如 `1.3.6.1.4.1.53184` 仅解析该范围的OID
4. 点击"解析MIB文件"

### 第二步: 查看解析结果
解析完成后右侧面板自动显示三个标签页:
- **静态指标**: 无索引的全局OID → 将生成 `<item>`
- **LLD自动发现**: 表格类OID按硬件分组 → 将生成 `<discovery_rule>` + `<item_prototype>`
- **触发器告警**: 所有health类指标自动关联的告警规则

每个指标自动分配了:
- 中文名称 (翻译词库)
- Zabbix标准化Key (如 `cpu.health`)
- 采集间隔 (1m / 5m / 1d)
- 值类型 (FLOAT / TEXT)
- 单位 (℃ / W / RPM / GB / %)
- 趋势保留 (30d / 0d)

### 第三步: 调整配置
在左侧"模板配置区"可修改:
- 模板前缀和型号
- SNMP团体字
- 三类指标的采集间隔
- 历史和趋势保留时间

### 第四步: 导出XML
1. 确认配置无误后，点击"下载XML文件"
2. 如需先检查，点击"预览XML"查看生成的XML内容
3. 系统自动校验XML结构完整性 (uuid唯一性、key格式、delay取值等)
4. 下载的XML可直接导入Zabbix 6.4

### 第五步: 导入Zabbix
1. 登录Zabbix Web
2. 配置 → 模板 → 导入
3. 选择下载的XML文件
4. 确认导入

## 翻译词库管理

访问 http://localhost:5173/translate

- 查看内置标准词库 (200+词条)
- 新增自定义翻译: 输入英文关键词 + 中文翻译
- 删除不需要的自定义词条
- 修改后重新解析MIB即可生效

## 历史任务管理

访问 http://localhost:5173/history

- 查看所有历史解析任务
- 复用: 加载历史任务的配置和解析结果
- 删除: 清理不需要的任务记录

## 异常处理

| 错误场景 | 处理方式 |
|---------|---------|
| MIB语法错误 | 前端弹窗提示具体错误行 |
| 缺失依赖MIB | 提示缺少的依赖文件名 |
| 文件格式不支持 | 提示仅支持 .mib / .txt |
| XML校验失败 | 列出具体错误项 (uuid重复/key格式等) |
| key含大写/中文 | 自动格式化修正为全小写 |
| delay非法值 | 自动修正为对应分类默认值 |
